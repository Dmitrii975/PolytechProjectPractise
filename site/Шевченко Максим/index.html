О проекте

Подробное описание системы автономного управления транспортными платформами, решаемых проблем и технологического стека

🎯 Проблематика

Существующие решения на рынке имеют ряд критических ограничений, препятствующих массовому внедрению автономных транспортных платформ:

01

Высокая задержка видео

Высокий уровень задержки при передаче видеоизображения между оператором и платформой.

02

Закрытые решения

Отсутствие или низкий уровень имеющихся на рынке решений с открытым исходным кодом.

03

Ограниченная совместимость

Текущее оборудование не может быть легко интегрировано с новыми системами и технологиями.

04

Нет мобильной поддержки

Многие существующие системы не адаптированы к мобильной среде или работе с облачными сервисами.

Цель проекта

Создание набора десктопного программного обеспечения и облачных сервисов для удалённого управления автономной транспортной платформой с минимальной задержкой и высокой надёжностью.

Задачи проекта

  • Реализация серверной и интерфейсной частей системы удалённого управления
  • Обеспечение защищённой системы авторизации и идентификации транспортных платформ и пульта ДУ
  • Оптимизация системы трансляции видео для работы между разными сетями
  • Разработка панели администратора для мониторинга и управления

Задача на текущий семестр

Модифицировать систему трансляции видео, так чтобы она передавала изображения с той же скоростью, но между машиной и клиентом оператора, которые находятся в разных сетях.

Журнал проекта

Хроника разработки системы автономного управления транспортными платформами

Сентябрь 2024 Новости

Старт проекта: формирование команды и определение архитектуры

Команда из 10 человек приступила к работе над системой автономного управления. Определены ключевые задачи: создание десктопного ПО, облачных сервисов и защищённой системы авторизации. Выбран технологический стек: C# для десктопного клиента, Node.js/Python для сервера, RTMP для видеопотока.

Проведена встреча с куратором Н. В. Панокиным, согласованы технические требования и сроки реализации.

Прогресс15%
Ноябрь 2024 Прогресс

Разработка системы передачи видео и принципа пулинга

Завершена базовая реализация системы передачи видеоизображения. Настроен H.265 кодек для эффективного сжатия потока. Внедрён принцип пулинга для безопасной проверки состояния подключения между оператором и платформой.

Созданы три версии интерфейса: начальный прототип, текущий рабочий вариант и специализированный интерфейс для тестирования. Админ-панель позволяет мониторить состояние всех подключённых платформ в реальном времени.

Прогресс55%
Январь 2025 Достижение

Успешное тестирование кросс-сетевой передачи видео

Команда завершила модификацию системы трансляции видео для работы между машиной и клиентом оператора, находящимися в разных сетях. Задержка видеопотока снижена до <80мс при сохранении качества изображения.

Развёрнуты все необходимые облачные сервисы. Система авторизации прошла аудит безопасности. Подготовлена документация и репозиторий проекта для открытого доступа.
